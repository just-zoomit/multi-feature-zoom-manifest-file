App manifests

App manifests are JSON files that contain the configuration for Zoom Marketplace Apps.

Manifest schema

Manifests are written in JSON using a specific structure.

Prerequisites

Before getting started, you will need the following:

  • Zoom has accepted you into the manifest beta program.
  • You have a Zoom account.
  • You are either the:
    • Zoom account owner, or
    • Account admin, or
    • Have been assigned the Zoom for developers role.
  • A development workspace where you have permissions to install apps.
  • You have experience creating apps in the Zoom Marketplace.
  • The app is a general app.

In this section, you'll create a user-managed Zoom Marketplace General App and configure it with Zoom App and Team Chatbot features.

Create your App: How to Create and Configure a Zoom App (User-Managed) with the Manifest API

  1. Visit the Zoom App Marketplace
    • Navigate to the Zoom App Marketplace and select General App.
  2. Start Building a New App
    • Click the "Develop" dropdown in the top-right.
    • Select "Build App".
  3. Choose App Type:
    • When prompted to choose an app type, select "General App" unless you have a specific use case.
    • Click "Create".
  4. Choose App Management Type
    • Select User-managed App. This allows individual users to install and authorize your app to access their Zoom data ‚Äî similar to apps accessing a user's Google Drive or Gmail
  5. Configure Basic App Information
    • On the "Basic Information" page:
    • Scroll down to the Redirect URL for OAuth section. For testing with Postman, add the Postman redirect URI:
https://oauth.pstmn.io/v1/callback
  1. Get App ID from URL:
  2. Make a PATCH Request (Test with Postman)
  • Now that your app is configured, you can test it by making a PATCH request to Zoom's API.
    • Use the PATCH method to update your app metadata, description, etc.
    • Method: PATCH
      • URL: https://api.zoom.us/v2/marketplace/apps/<APP_ID>
      • Headers:
      • Authorization: Bearer <YOUR_ACCESS_TOKEN>
      • Content-Type: application/json
      • Body:
